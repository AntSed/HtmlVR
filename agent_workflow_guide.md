# HtmlVR: AI-Agent Video Production Workflow Guide

This guide describes the 3-step production pipeline for generating voice narration, chalkboard illustrations, and compiling them into a synchronized multi-track sequence with simple, fun visual overlays.

---

## Шаг 1. Генерация звука (Audio Generation)

Для генерации голоса диктора используется скрипт `generate_voice.py`.

1. **Команда вызова**:
   Каждая часть текста генерируется с помощью вызова:

   ```bash
   python generate_voice.py --text "<Текст сегмента>" --output "<Путь к WAV файлу>" --voice Sadaltager --preset podcast --style-notes "<Настройки стиля>" --pace-notes "<Настройки темпа>"
   ```

2. **Параметры стиля и темпа (дословно)**:
   * `--style-notes`: `"Стиль: Профессиональный диктор научно-популярных новостей (news anchor) с теплой, доброжелательной подачей. Тон уверенный, авторитетный, но близкий и располагающий, с мягкой улыбкой в голосе. Артикуляция: Мягкая, естественная, без нарочитого выговаривания слогов. Полностью исключи любые заминки, вздохи и запинки импровизации."`
   * `--pace-notes`: `"Уверенный, умеренно-беглый и чёткий темп речи (без спешки, но и без медленных театральных пауз между словами). Слова плавно перетекают одно в другое, формируя единую цельную линию повествования."`
3. **Определение таймингов**:
   Длительность полученного аудиофайла считывается с помощью стандартного модуля `wave` в Python (`frames / framerate`) и используется для точного позиционирования клипов на таймлайне.

## Шаг 2. Генерация картинок (Image Generation)

Для иллюстраций используется локальный прокси-сервер генерации картинок (`http://127.0.0.1:8000/v1/images/generations`).

### Ключевые правила генерации иллюстраций

1. **Пошаговость и фокус**: Иллюстрации генерируются **последовательно, по одной за раз, без спешки**. Это необходимо, чтобы детально продумать соответствие визуальных метафор тексту каждого конкретного абзаца.
2. **Юмор и кастомные детали**: Иллюстрации не должны быть сухими абстрактными схемами. При любой возможности добавляйте в промпт забавные, ироничные детали.
3. **Единый стиль**: Всегда запрашивайте простой "детский" рисунок белым мелом на фоне школьной доски (`Super simple child-like 2D white line drawing`, `extremely primitive chalkboard sketch`, `raw line art`, `hand drawn look`).

## Шаг 3. Компоновка таймлайна и простые анимации

> [!IMPORTANT]
> **Текстовые плашки (карточки с названиями и описаниями) НЕ используются.** Они перегружают экран, закрывают красивые иллюстрации и отвлекают зрителя от диктора. Оверлей должен содержать исключительно прозрачную холстовую анимацию для оживления рисунка.

### Рекомендуемый воркфлоу: Один ассет — один клип

Для максимальной надежности и скорости работы агент **должен генерировать каждый анимированный элемент в виде отдельного HTML-файла с прозрачным фоном** (один ассет = один клип на отдельной дорожке).

#### Почему подход «Один ассет — один клип» намного удобнее для агента

1. **Простота и стабильность кода**: Написать простой HTML/Canvas скрипт для одного конкретного ассета (например, только анимированные глаза или только пульсирующие стрелки) модель Pro-pool может с первой попытки без ошибок. Генерация сложного комбайна с десятком разнородных элементов часто приводит к багам.
2. **Отсутствие риска при разделении**: Сначала написать всё в одном файле, а затем пытаться разрезать его скриптом на части — крайне ненадежный шаг для AI-агента. Это часто ломает циклы анимаций (GSAP / requestAnimationFrame), сбивает относительные координаты и крашит JS-код.
   > [!TIP]
   > Если агент всё же нарушил правило и сгенерировал несколько ассетов внутри одной анимации, воспользуйтесь скриптом [split_composition.py](file:///c:/HtmlVR/split_composition.py) для автоматического разделения на одиночные клипы.
3. **Отказ от ручного вычисления координат**: Благодаря встроенным в HtmlVR инструментам перемещения, масштабирования и поворота в реальном времени, агенту **больше не нужно пытаться рассчитать точные координаты наложения в коде**. Агент генерирует элемент в стандартном разрешении `1920x1080`, а человек в браузере визуально подгоняет его по месту за пару секунд.
4. **Гибкость на таймлайне**: Разные элементы (глаза, выноски, спецэффекты) лежат на отдельных дорожках. Их можно двигать по времени независимо, растягивать, обрезать, добавлять индивидуальные переходы и регулировать прозрачность.

Каждый фрагмент собирается на таймлайне из параллельных слоев:

* **Track 4 (Background)**: Статичное изображение (`.png`) сгенерированной иллюстрации.
* **Track 1, 2, 3 (Animation Overlays)**: Прозрачные HTML-клипы с анимациями ассетов.
* **Track (Audio)**: WAV-файл озвучки сегмента.

### Правила анимации оверлея

* **Юмор и контекст**: Делайте анимации забавными (например, вылетающие из ушей облачка пара, летящие смайлики-молекулы или колеблющиеся звуковые волны из рупора).

Все треки позиционируются последовательно на основе длительности аудиофайлов, записываются в `project_state.json` и отправляются методом POST на REST API `/api/project/state?project=<название_проекта>` для мгновенной синхронизации с редактором и рендером.
Если аудиофайл длинный, для него можно сгенерировать две картинки, чтобы зритель не заскучал, на твоё усмотрение.

## Полезные ссылки и контекст

Для качественного выполнения всех этапов агенту рекомендуется использовать следующие справочные материалы:

1. **Архитектура HtmlVR и REST API**: Детальное описание работы видеосеквенсора, доступных CLI-команд, REST API и логики вложенных проектов находится в файле [gemini.md](file:///c:/HtmlVR/gemini.md).
2. **Разделение оверлеев**: Скрипт [split_composition.py](file:///c:/HtmlVR/split_composition.py) автоматически нарезает комбинированный HTML-клип на независимые файлы по границам логических блоков (определяется пустыми строками и комментариями вида `// Comment`), отключает контейнерный зум/панорамирование, сохраняет общую разметку и LCG-генератор. Применяется, если оверлей сгенерирован единым файлом с несколькими анимированными деталями.

## Размещение скриптов сборки проектов

> [!IMPORTANT]
> **Скрипты сборки конкретных проектов и таймлайнов (например, `build_genius_stickman_partX.py`) НЕ должны находиться в корневой директории репозитория.**
> Все подобные скрипты сборки должны сохраняться и запускаться из папки самого проекта:
> `public/projects/<название_проекта>/` или `public/projects/<название_проекта>/build/`.
> В корне проекта разрешено находиться только общесистемным утилитам (`generate_voice.py`, `monitor_trigger.py`, `agent_director.py`, `watermark_cleaner.py`, `split_composition.py`).
